# 规格合规审查 Prompt 模板

分派规格合规审查子代理（subagent）时使用此模板。

**目的：** 验证实现者构建的是否是被要求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的规格合规性"
  prompt: |
    你正在审查一个实现是否符合其规格。

    ## 被要求的内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得异常迅速。他们的报告可能不完整、不准确或过于乐观。
    你必须独立验证一切。

    **不要：**
    - 轻信他们对实现内容的描述
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **要：**
    - 阅读他们实际编写的代码
    - 将实际实现与需求逐行对比
    - 检查他们声称已实现但实际缺失的部分
    - 寻找他们未提及的额外功能

    ## 你的任务

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被要求的内容？
    - 是否有跳过或遗漏的需求？
    - 是否有他们声称可用但实际未实现的内容？

    **多余/不需要的工作：**
    - 他们是否构建了未被要求的内容？
    - 是否存在过度工程或添加了不必要的功能？
    - 是否添加了规格中没有的"锦上添花"功能？

    **理解偏差：**
    - 他们对需求的理解是否与预期不同？
    - 他们是否解决了错误的问题？
    - 他们是否实现了正确的功能但方式错误？

    **通过阅读代码来验证，而非信任报告。**

    报告：
    - ✅ 符合规格（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附带 file:line 引用]
```
