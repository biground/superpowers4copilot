# KiloCode 支持实施计划

**目标：** 让 superpowers4copilot 支持 KiloCode，通过安装脚本将 agents、commands、skills 复制到 KiloCode 全局配置目录。

**策略：** 在现有安装脚本中添加 `--kilocode` 参数，复制时自动转换 Copilot 格式为 KiloCode 格式（frontmatter 转换 + 工具名称替换），并创建 KiloCode 工具映射参考文件。

---

## 2. 创建 KiloCode 格式的 Agent 文件

**新建目录：** `agents-kilocode/`

为每个现有 agent 创建 KiloCode 格式版本。转换规则：

### Frontmatter 转换

| Copilot 字段 | KiloCode 字段 | 转换规则 |
|---|---|---|
| `name: gem-orchestrator` | 文件名 `gem-orchestrator.md` | name → 文件名 |
| `description: \| ...` | `description: ...` | 保持不变 |
| `user-invocable: true` | `mode: primary` 或 `mode: all` | 见下表 |
| `user-invocable: false` | `mode: subagent` | 不可直接调用 |
| `disable-model-invocation: true` | (移除) | KiloCode 无此字段 |

Mode 映射：
- `gem-orchestrator` → `mode: primary`（主入口，用户可直接 @）
- `gem-planner`, `gem-implementer`, `gem-reviewer`, `gem-researcher`, `gem-critic`, `gem-designer`, `gem-browser-tester`, `gem-documentation-writer` → `mode: all`（可主叫也可子代理）
- `security-reviewer`, `performance-optimizer`, `a11y-architect`, `silent-failure-hunter`, `code-reviewer` → `mode: all`
- `se-technical-writer`, `narrative-writer` → `mode: all`
- `global.instructions.md` → 不作为 agent，内容合并到 AGENTS.md

### Body 工具名称替换

安装脚本在复制时执行以下替换：

| Copilot 工具名 | KiloCode 工具名 |
|---|---|
| `runSubagent` | `task` |
| `vscode_askQuestions` | `question` |
| `manage_todo_list` | `todowrite` |
| `run_in_terminal` | `bash` |
| `read_file` | `read` |
| `create_file` | `write` |
| `replace_string_in_file` | `edit` |
| `multi_replace_string_in_file` | `edit` |
| `grep_search` | `grep` |
| `file_search` | `glob` |
| `fetch_webpage` | `webfetch` |
| `get_errors` | `bash` (配合 lint 命令) |
| `semantic_search` | `grep` |
| `list_dir` | `read` |
| `view_image` | `read` |
| `vscode_listCodeUsages` | `grep` |

### Command Frontmatter 格式

```yaml

---
```

### Command Body 转换

- 工具名称替换（同 agent 的替换表）
- `skills/` 路径引用保持不变（KiloCode 的 skill 工具会解析路径）
- 移除 "VS Code Copilot Chat 工具映射参考" 等平台特定引用，改为 KiloCode 工具映射

## 5. 修改安装脚本

### install.ps1 修改

添加 `-Kilocode` 参数：

```powershell
param(
    [switch]$Insiders,
    [switch]$Uninstall,
    [switch]$Kilocode
)
```

当 `$Kilocode` 为 true 时：

1. **目标目录变更：**
   - agents → `$env:USERPROFILE\.kilo\agent\`
   - commands → `$env:USERPROFILE\.kilo\command\`
   - skills → `$env:USERPROFILE\.kilo\skills\`
   - 全局指令 → `$env:USERPROFILE\.kilo\AGENTS.md`

2. **复制来源变更：**
   - agents 从 `agents-kilocode\` 目录复制
   - commands 从 `commands-kilocode\` 目录复制
   - skills 从 `skills\` 目录复制（格式兼容，无需转换）
   - AGENTS.md 从 `agents-kilocode\AGENTS.md` 复制

3. **卸载逻辑：** 删除对应目录下安装的文件

4. **输出信息变更：** 显示 KiloCode 相关的验证和使用说明

### install.sh 修改

同步添加 `--kilocode` 参数，逻辑与 PowerShell 版本一致：

- 目标目录：`$HOME/.kilo/agent/`、`$HOME/.kilo/command/`、`$HOME/.kilo/skills/`
- 复制来源同上
- 卸载逻辑对应

### 参数互斥

`-Insiders` 和 `-Kilocode` 互斥（安装到不同平台），使用时需二选一。

## 6. 更新 README.md

在"安装"章节下添加 KiloCode 安装说明：

```markdown
### KiloCode 安装

**1. 克隆仓库**（同上）

**2. 运行安装脚本**

macOS / Linux:

```bash
cd ~/.copilot/superpowers4copilot
./install.sh --kilocode
```

Windows (PowerShell):

```powershell
cd $HOME\.copilot\superpowers4copilot
.\install.ps1 -Kilocode
```

安装脚本会将 agents、commands 和 skills 复制到 KiloCode 全局配置目录（`~/.kilo/`）。

### 卸载

```bash
# macOS / Linux
./install.sh --kilocode --uninstall

# Windows
.\install.ps1 -Kilocode -Uninstall
```

### 验证安装

在 KiloCode 中输入 `@gem-orchestrator 你好`，如果代理能正常响应并进入阶段检测，说明安装成功。
```

同时在代理系统、Slash 命令等章节补充 KiloCode 下的触发方式说明。
